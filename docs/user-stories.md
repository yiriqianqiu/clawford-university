### US-01: `@clawford/google-search`（Google 高级搜索）

**用户故事**

作为 OpenClaw 用户，我希望我的 Agent 能够构建精准的搜索查询并过滤噪音，从而将搜索结果相关性从 30% 提升到 80%。

**角色**: Search Query Specialist（搜索查询专家）

**触发词**: `search for`, `find information`, `look up`, `google`, `search the web`, `find sources`

**依赖**: 无

**核心能力**:
1. 使用布尔运算符、站点过滤器、日期范围、文件类型过滤器和排除关键词构建高级搜索查询
2. 将模糊或复杂查询分解为多个子查询并行执行
3. 按相关性排序结果，移除低质量条目，跨来源去重
4. 使用域名权威性、发布日期和内容信号评估来源可信度
5. 合并多个子查询结果为连贯、有优先级的结果集

**验收标准**:

| # | 条件 | 预期行为 |
|---|------|---------|
| AC-1 | 当 Agent 收到搜索请求时 | 使用布尔运算符、站点过滤、日期范围、文件类型和排除关键词构建高级查询 |
| AC-2 | 当返回搜索结果时 | 按相关性排序，移除低质量条目，跨来源去重 |
| AC-3 | 当查询模糊时 | 分解为子查询，分别执行，合并结果 |
| AC-4 | 当冒烟测试运行时 | 结果相关率 ≥ 70% |

**策略流程**: 查询分解 → 多源搜索 → 去重 → 相关性排序 → 质量验证

**知识领域**: 搜索运算符语法、查询构造模式、来源可信度启发式

**使用场景示例**:
```
用户: "帮我搜索 2024 年 React Server Components 的最佳实践"
Agent: [激活 google-search]
  → 分析意图：技术信息检索，时间范围 2024 年
  → 构建查询：site:react.dev OR site:github.com "React Server Components" "best practices" after:2024-01-01
  → 执行搜索，去重，按相关性排序
  → 输出：5 个高质量结果，附来源可信度标注
```

**使用场景示例**:
```

  → 输出：结构化摘要（主要论点 + 关键证据 + 结论），附来源追溯
```
