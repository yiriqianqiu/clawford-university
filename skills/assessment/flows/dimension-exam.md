# Dimension Exam Flow

## 1. Session Setup

```
sessionId  = "exam-{YYYYMMDD}-{HHmm}"
dimension  = D{N}  (1–5, identified from user intent)
totalQ     = 1  (randomly selected)
resultFile = "results/exam-{sessionId}-d{N}.md"
userLang   = detected from activation message
```

### Dimension Detection

| User says | Dimension |
|-----------|-----------|
| D1 / reasoning / planning / 知识与推理 / 推理 | D1 |
| D2 / retrieval / search / 信息检索 / 检索 | D2 |
| D3 / creation / writing / 内容创作 / 写作 | D3 |
| D4 / execution / code / build / 执行与构建 / 代码 | D4 |
| D5 / orchestration / tools / 工具编排 / 工具 | D5 |

If unclear, ask in the user's detected language (offer 5 dimension options).

## 2. Pre-exam Announcement

Output in user's detected language:

```
# D{N} {Dimension Name} — Single Dimension Assessment

Session: exam-{ID}
Dimension: D{N} {Dimension Name}
Questions: 1 (randomly selected from Easy/Medium/Hard)

📋 Rules:
- Answer is SUBMITTED immediately — no revision
- I must complete this independently — no user assistance
- Missing tools = auto-skip with score 0

🎲 Random selection: {difficulty} ×{multiplier}

Starting now...
```

## 3. Question Selection & Execution

```
1. RANDOM SELECT one of Q1-EASY, Q2-MEDIUM, Q3-HARD from the dimension file

2. PRE-CHECK tool dependencies:
   → IF missing required tool:
     OUTPUT skip notice, score=0, save report, STOP

3. EXECUTE (EXAMINEE): Answer genuinely, no rubric, no user help

4. SCORE (EXAMINER): Apply rubric strictly, -5% correction

5. IMMEDIATELY OUTPUT the Question Card to user (FINAL SUBMISSION)
```

## 4. Dimension Score Calculation

```
# Single question — score IS the dimension score:
DimScore_raw = RawScore of the selected question
DimScore_adj = AdjScore of the selected question
# (If skipped: DimScore = 0)
```

## 5. Report Template

Save to `results/exam-{sessionId}-d{N}.md`:

```markdown
# D{N} {Dimension Name} — Single Dimension Assessment Report

**Session**: exam-{ID}
**Timestamp**: {datetime}
**Dimension**: D{N} {Dimension Name} (weight: {weight}%)
**Question**: {difficulty} ×{multiplier} (randomly selected)

---

## Dimension Score

| Metric | Raw | Adjusted (-5%) |
|--------|-----|----------------|
| {Dimension Name} | {dim_raw}/100 | {dim_adj}/100 |
| Performance Level | {level_raw} | {level_adj} |
| Status | {✅ Answered / ⏭️ Skipped} | |

---

## Question Detail

[The submitted question card]

---

## Diagnosis & Suggestions

**Performance analysis**: {analysis}
**Improvement suggestion**: {suggestion}

---

*Generated by clawford-assessment v2.0 | {timestamp}*
*📝 Tested with 1 random question — re-run for different difficulty.*
```

Translate labels into user's detected language when presenting.

## 6. Update Index

Append to `results/INDEX.md`:

```
| {date} | {sessionId} | D{N} {Dimension Name} ({difficulty}) | {dim_adj} | {level} | — |
```

## 7. Completion Announcement

Output in user's detected language:

```
D{N} {Dimension Name} assessment complete! ✅

Question difficulty: {difficulty} ×{multiplier}
Dimension score: {dim_raw}/100  (adjusted: {dim_adj}/100)
Performance level: {level}
Status: {✅ Answered / ⏭️ Skipped}

Result saved to: results/exam-{sessionId}-d{N}.md
Say "full exam" to run the complete 5-dimension assessment.
Say "D{N} test" again for a different random question.
```
