# Phase 3 — Report Generation

Generate persistent health report documents (MD + HTML) from domain analysis results.

## 3.1 — Output Location

```
$OPENCLAW_HOME/memory/health-reports/
```

Create the directory if it does not exist.

## 3.2 — File Naming

Both files share the same timestamp-based name:

```
healthcheck-YYYY-MM-DD-HHmmss.md
healthcheck-YYYY-MM-DD-HHmmss.html
```

Example: `healthcheck-2026-03-04-153012.md` / `healthcheck-2026-03-04-153012.html`

## 3.3 — Markdown Report Content

The `.md` file must include all of the following sections in order:

```markdown
# OpenClaw Health Report

> Generated: YYYY-MM-DD HH:mm:ss
> Mode: [Full Check / Targeted: <domain>]

## Overview

OpenClaw Health: [X] pass [X] warn [X] error — Overall: [status summary in REPORT_LANG]

## Domain Scores

| Domain | Status | Score | Summary |
|--------|--------|-------|---------|
| [Hardware]  | [STATUS] | XX/100 | [summary] |
| [Config]    | [STATUS] | XX/100 | [summary] |
| [Security]  | [STATUS] | XX/100 | [summary] |
| [Skills]    | [STATUS] | XX/100 | [summary] |
| [Autonomy]  | [STATUS] | XX/100 | [summary] |

## Issue List

(Copy L2 issue table from domain results; if all pass, write "No issues found" in REPORT_LANG)

## Fix Recommendations

(For each issue: one-liner fix command + brief explanation; if none: "No fixes needed" in REPORT_LANG)

## Environment Snapshot

- OS: [platform] [arch] [version]
- Node.js: [version]
- OpenClaw: [version] ([commit])
- Memory: XX.X GB / XX.X GB (XX%)
- Disk: XX.X GB / XX.X GB (XX%)
- Skills: [X] installed, [X] botlearn

---
*Generated by @clawford/healthcheck*
```

## 3.4 — HTML Report Content

The `.html` file is a **self-contained, styled HTML page** suitable for browser viewing.
Use `scripts/generate-report.sh --format html` as the base renderer, or generate inline if
the analysis JSON is not available. The HTML must include:

- Dark-theme styled layout (consistent with existing `generate-report.sh` output)
- Overall status banner with color-coded score
- Domain grid cards with traffic-light colors
- Issue table (if any warnings or errors)
- Environment snapshot section
- Responsive design + print-friendly `@media print` rules
- Footer: `Generated by @clawford/healthcheck · YYYY-MM-DD HH:mm:ss`

## 3.5 — Generation Protocol

1. After Phase 2 domain analysis, construct the report data from the domain results already in memory.
2. Run `scripts/generate-report.sh --input <analysis.json> --format all --output-dir $OPENCLAW_HOME/memory/health-reports/` if analysis JSON was saved, **or** write both files directly using the domain result data.
3. Rename the output files to match the `healthcheck-YYYY-MM-DD-HHmmss.*` naming convention.
4. Verify both files exist and are non-empty.
5. Report to the user (in REPORT_LANG):

```
Report archived:
  MD:   $OPENCLAW_HOME/memory/health-reports/healthcheck-<timestamp>.md
  HTML: $OPENCLAW_HOME/memory/health-reports/healthcheck-<timestamp>.html
```

If HTML generation fails (e.g., Node.js issue), still produce the MD file and note the HTML failure.
